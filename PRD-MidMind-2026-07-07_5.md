# PRD: MidMind — Dota 2 Mid Lane Assistant

**Версия:** 1.2
**Дата:** 2026-07-07
**Статус:** Draft для передачи в разработку (AI-assisted development через Claude)

---

## 1. Обзор приложения и цели

### 1.1 Описание
MidMind — desktop-приложение для Windows с игровым оверлеем поверх Dota 2, которое помогает игрокам средней линии (mid lane) на всех этапах матча: от выбора героя до принятия решений в реальном времени. Приложение использует только официальные и легальные источники данных: Game State Integration (GSI) API от Valve и STRATZ GraphQL API.

### 1.2 Цели продукта
1. Повысить винрейт и качество принятия решений мидеров среднего рейтинга.
2. Дать структурированный план на каждый матч: пик → лайнинг → тайминги → мид-гейм.
3. Работать ненавязчиво: минимум визуального шума, подсказки только в нужный момент.
4. Оставаться на 100% в рамках правил Valve (никакого чтения памяти игры, автоматизации действий, снятия тумана войны).

### 1.3 Платформа отображения
Игра запускается в режиме **borderless windowed**. Оверлей — прозрачное always-on-top окно поверх игры: по умолчанию click-through (клики проходят сквозь него в игру), интерактивность включается хоткеем для работы с расширенной панелью. Инъекции в процесс игры не используются.

**Предпосылка:** стабильная работа Dota 2 в borderless на целевой машине. Проверяется и при необходимости настраивается первым шагом M0 (Fullscreen Optimizations, Vulkan/DirectX 11, G-Sync/V-Sync, лимит FPS); критерий — стабильный FPS в 2–3 тестовых матчах.

**Fallback:** режим второго монитора — то же приложение в обычном окне вместо оверлея, с опциональными звуковыми оповещениями.

### 1.4 Модель использования
Личное приложение для одного пользователя, устанавливается и работает на его машине. Публичное распространение не планируется. Логика подсказок изолируется в отдельный модуль (`advice-engine`) для удобства тестирования и развития.

## 2. Пользователь

**Единственный пользователь:** владелец приложения — игрок mid lane с рейтингом в диапазоне Archon–Ancient (~2500–5000 MMR).

Требования к подаче, вытекающие из уровня игры:
- Базовая механика известна; банальные советы («купи бутылку») не нужны и раздражают.
- Ценность — в системности: нюансы матчапов, тайминги силы, дисциплина решений.
- Цифры вызывают доверие: винрейты, статистика, данные меты.
- Оверлей не должен быть перегружен во время матча.

**Персонализация:** Steam ID пользователя задаётся один раз в настройках (или автоопределяется из GSI) и даёт личную статистику: «У тебя 38% винрейта против Viper за последние 50 игр — вот план».

## 3. Основные функции и функциональность

### F1. Помощник по пикам (Draft Assistant)

**Описание:** На стадии драфта приложение анализирует ВСЕ открытые пики вражеской команды и пики союзников (через GSI или ручной ввод) и рекомендует героя на мид с учётом контрпиков против всего вражеского драфта и синергии с союзниками.

**Два режима ранжирования (переключатель в панели драфта):**
- **Meta-режим:** чистая статистика меты, личная статистика не учитывается. Для случаев «хочу объективно лучший пик» или когда Steam ID не привязан.
- **Personal-режим:** к статистике меты добавляется личный винрейт и пул героев пользователя.

**Функциональность:**
- Автоопределение стадии драфта и всех открытых пиков обеих команд через GSI (`map.game_state = DOTA_GAMERULES_STATE_HERO_SELECTION`).
- Скоринг кандидата против всего вражеского драфта: `counter_score` = взвешенное среднее винрейтов матчапов кандидата против каждого открытого врага (вес соперника по миду ×2, остальные ×1; веса конфигурируемы).
- Синергия с союзниками: `synergy_score` = среднее значение синергии кандидата с открытыми пиками своей команды (STRATZ отдаёт данные «with» аналогично «vs»).
- Итоговая формула: `score = w1 * counter_score + w2 * synergy_score + w3 * personal_winrate`; в Meta-режиме `w3 = 0`. Дефолтные веса: w1=0.5, w2=0.4, w3=0.1 (конфигурируемы).
- Пересчёт рекомендаций в реальном времени при каждом новом открытом пике любой из команд.
- Прозрачность в UI: у каждого кандидата видна разбивка — против кого из открытых врагов он хорош/плох и с кем из союзников синергия, а не только итоговая цифра.
- В Personal-режиме приоритет героям, которых пользователь реально играет (≥10 матчей за последние N патчей).
- Ручной режим: выбор героев врага/союзников вручную, если автоопределение недоступно.

**Acceptance criteria:**
- [ ] При входе в стадию пика оверлей автоматически показывает панель драфта ≤ 2 сек после события GSI.
- [ ] Рекомендации пересчитываются ≤ 2 сек после каждого нового открытого пика любой из команд.
- [ ] Переключатель Meta/Personal меняет ранжирование мгновенно (оба скоринга рассчитываются заранее).
- [ ] Для каждого кандидата отображается детализация: винрейт против каждого открытого врага + синергия с союзниками + размер выборки.
- [ ] Данные соответствуют текущему патчу; при недоступности STRATZ показывается кэш с пометкой давности.

**Technical considerations:** матчап-данные кэшируются локально (SQLite/JSON) с TTL 24 ч; прогрев кэша по топ-40 мид-героев меты при старте приложения, чтобы уложиться в лимиты STRATZ.

### F2. Гайд на лайнинг (Lane Plan)

**Описание:** После подтверждения пиков приложение генерирует «план на лайн» для конкретной ПАРЫ героев: твой герой × вражеский мидер. Все советы пишутся с позиции твоего героя — никаких рекомендаций, которые твой герой физически не может исполнить.

**Функциональность:**
- Стартовая закупка и скиллбилд: популярные + высоковинрейтные билды из STRATZ именно для пары «твой герой vs вражеский».
- Карточка матчапа (ключ — пара героев, а не один враг): кто сильнее на каких уровнях в ЭТОЙ паре, окна агрессии твоего героя, окна опасности от врага, чего опасаться, окна убийства.
- Пример различия: против Viper карточка для Storm Spirit («играй от опыта до 6, не разменивайся») и для Huskar («агрессируй с 1 уровня — Viper не перебивает твой размен») принципиально разные, хотя враг один и тот же.
- План таймингов: когда бутылка, когда первый выход силы, целевой тайминг ключевого предмета.
- Контент матчапов: гибрид — статистика из STRATZ + локальная база знаний (curated JSON с заметками по топ-матчапам мида), редактируемая без перекомпиляции.

**Acceptance criteria:**
- [ ] План на лайн доступен в расширенной панели ≤ 5 сек после финализации пиков.
- [ ] Карточки матчапов заполнены для наиболее частых ПАР из топ-20 мид-героев меты, написаны с позиции твоего героя (минимум: 3 тезиса «что делать», 2 тезиса «чего бояться», тайминги силы обеих сторон). Приоритет заполнения — по частоте встречаемости пары.
- [ ] Для пар вне базы знаний показывается статистический fallback (билды, винрейт пары, пики силы из hero-profiles) — без текстов, которые могли бы не подходить твоему герою.
- [ ] Скиллбилд и стартовая закупка соответствуют текущему патчу.

### F3. Тайминговые напоминалки (Timing Alerts)

**Описание:** Всплывающие уведомления и таймеры на основе игрового времени из GSI.

**События MVP:**
| Событие | Логика | Уведомление |
|---|---|---|
| Руны воды | 2:00 и 4:00 | за 20 сек |
| Руны силы | каждые 6:00 с 6:00 | за 30 сек |
| Руны богатства | 0:00, далее каждые 3:00 | за 20 сек |
| Стак кемпа | xx:53 | за 10 сек |
| Ночь/день | каждые 5:00 | за 15 сек |
| Аванпосты XP | каждые 7:00 | за 30 сек |
| Tormentor | 20:00 (по актуальному патчу) | за 60 сек |
| Смерть → тайминг байбека | из GSI `hero.buyback_cooldown` | по факту |

- Все тайминги вынесены в конфиг (`timings.json`), т.к. Valve меняет их патчами.
- Каждый тип уведомления отключается в настройках отдельно.
- Умное подавление: не показывать напоминание о стаке, если герой мёртв или идёт активный файт (эвристика: потеря >30% HP за 3 сек).

**Acceptance criteria:**
- [ ] Точность срабатывания ±1 сек от игрового времени (GSI шлёт обновления ~2 раза/сек).
- [ ] Уведомление живёт на экране 5–8 сек и исчезает без действий пользователя.
- [ ] Одновременно на экране не более 2 уведомлений (очередь с приоритетами).
- [ ] Все тайминги редактируются в локальном конфиге и применяются без пересборки приложения.

### F4. Ситуативные подсказки (Situational Advice) — rule-based

**Описание:** Подсказки на основе состояния игры из GSI. **Только данные, доступные легально:** свой герой (HP/мана/уровень/способности/предметы/золото), время игры, счёт, K/D/A, состояние карты. GSI **не** даёт позиции и предметы врагов — логика строится на своих данных и таймингах.

**Ключевой принцип: правила герой-зависимы (hero-aware).** Универсальное правило «ульта готова = окно агрессии» неверно: ультимейт Storm Spirit или Puck — это окно убийства, а ультимейт Templar Assassin (Psionic Trap) сам по себе агрессии не даёт. Каждое правило параметризуется профилями героев — и твоего, и вражеского.

**Профили героев (`hero-profiles.json`):** для каждого мид-героя задаются флаги и параметры: `ult_is_kill_window` (bool), `power_spike_levels` (например, [6, 7] у Storm Spirit; [2, 3] у Shadow Fiend), `aggression_pattern` (all-in / trade / passive-farm), `typical_level6_time_sec` (медиана из статистики). Правила ссылаются на эти поля вместо жёстко зашитых условий.

**Модель врага (важное ограничение GSI):** GSI не отдаёт уровень и способности вражеских героев. Уровень вражеского мидера ОЦЕНИВАЕТСЯ по игровому времени и статистическим таймингам его героя (`typical_level6_time_sec`). Все подсказки об угрозе от врага помечаются как оценка («Puck, вероятно, взял 6») и калибруются консервативно, чтобы не давать ложной уверенности.

**Примеры правил MVP (движок правил, приоритеты, cooldown на повтор):**
- `hp < 35% && mana < 20%` → «Отойди на хил / используй бутылку» (не чаще 1 раза/60 сек).
- `ultimate.ready && my_hero.ult_is_kill_window && power_rune_timing ± 30 сек` → «Ульта готова + скоро руна — окно агрессии». Для героев с `ult_is_kill_window = false` (TA, Invoker и т.п.) правило не срабатывает либо заменяется героя-специфичной версией.
- `estimated_enemy_level >= 6 && enemy_hero.ult_is_kill_window` → «⚠ У [врага], вероятно, готова ульта — не стой на убиваемом HP» (одноразово, в момент оценённого тайминга).
- `gold > 1200 && !tp_scroll` → напоминание про TP (для опытных — отключаемо).
- `deaths >= 3 && game_time < 12:00` → смена плана: «Играй от опыта, не контести руны без миссинга» (карточка «как играть из-за спины» для текущей пары героев).
- `level in my_hero.power_spike_levels && matchup.kill_window_at_level` → «Твой пик силы в этом матчапе — ищи килл или мув с руной».

**Acceptance criteria:**
- [ ] Движок правил читает декларативный конфиг (JSON/YAML: условие → сообщение → приоритет → cooldown).
- [ ] Задержка от изменения состояния GSI до показа подсказки ≤ 1 сек.
- [ ] Ни одно правило не спамит: глобальный лимит — не более 1 ситуативной подсказки в 30 сек.
- [ ] Новые правила добавляются без изменения кода движка.
- [ ] Правила параметризованы профилями героев: одно и то же событие (готовность ульты, достижение уровня 6) даёт разные подсказки для разных пар героев или не срабатывает вовсе.
- [ ] Подсказки, основанные на оценке состояния врага, визуально помечены как оценка («вероятно»), а не подаются как факт.

**Technical considerations:** это модуль `advice-engine` — проектируется как чистая функция `(gameState, matchupContext, personalStats) → advice[]`, без зависимостей от UI и Electron API: так его проще тестировать и при необходимости перенести на сервер.

### F5. Оверлей: три режима отображения

1. **Компактная панель (постоянная, угол экрана):** конструктор из НАСТРАИВАЕМЫХ статичных виджетов. Дефолтный набор: таймер ближайшего события, фаза игры, индикатор ближайшей руны. Размер панели адаптируется к набору виджетов, панель перетаскивается, позиция запоминается.

   **Конструктор виджетов (меню в настройках):** пользователю доступен ПОЛНЫЙ каталог данных, которые отдаёт GSI, сгруппированный по категориям — Герой (HP, мана, уровень, кулдаун байбека…), Игрок (золото, GPM, XPM, K/D/A, ластхиты/денаи…), Матч (игровое время, счёт, день/ночь…), Способности и Предметы. Любое поле включается как виджет на панели: чекбокс + drag-and-drop порядок. Составные виджеты (таймер события, индикатор руны, счётчик стаков) доступны в том же меню как пресеты.

   Каталог полей описывается конфигом `gsi-field-catalog.json` (см. раздел 5.2) и наполняется по результатам ревизии GSI в M0 — в меню попадают только поля, реально приходящие в пакетах.

   **Пресет позиции «Вместо стандартной панели»:** панель MidMind размещается точно поверх стандартной панели статистики Dota (левый верхний угол: KDA, ластхиты/денаи, GPM/XPM), визуально заменяя её. Важно: родной HUD игры скрыть нельзя (легального API нет) — панель именно накрывает его; координаты пресета калибруются под разрешение экрана. Дефолтный набор виджетов пресета повторяет стандартную панель (LH/DN, GPM/XPM, KDA из GSI) и расширяется любыми полями каталога.

   **Бенчмарк-виджеты (в стиле Dota Plus):** сравнение текущих показателей с эталоном для твоего героя на текущей минуте — «LH: 45 / норма 52», «Networth: −800 от нормы». Эталонные кривые (медиана и 75-й перцентиль по рангу Archon–Ancient) предрассчитываются из статистики STRATZ/OpenDota в конфиг `benchmarks.json` и обновляются с патчами. Цветовая индикация: зелёный — опережаешь, жёлтый — в норме, красный — отстаёшь. Доступны в конструкторе как пресеты: LH, networth, XP.
2. **Всплывающие уведомления:** появляются в заданной зоне (не перекрывая миникарту и HUD Dota), auto-dismiss 5–8 сек.
3. **Расширенная панель (по хоткею, по умолчанию F9):** план на лайн, карточка матчапа, билд, личная статистика матчапа. Открывается/закрывается тем же хоткеем; предполагаемые моменты использования — пауза, смерть, ожидание крипов.

**Acceptance criteria:**
- [ ] Оверлей отображается поверх Dota 2 в borderless windowed; по умолчанию click-through (не перехватывает клики), интерактивность включается/выключается хоткеем.
- [ ] FPS-просадка от оверлея ≤ 5% (замер на референсной машине).
- [ ] Все элементы оверлея скрываются одним «тихим режимом» (хоткей).
- [ ] Позиции и видимость элементов сохраняются между сессиями.
- [ ] Меню конструктора показывает все поля из gsi-field-catalog.json, сгруппированные по категориям; включение/выключение и порядок применяются к панели без перезапуска приложения.
- [ ] Добавление нового поля в каталог (конфиг) делает его доступным в меню без изменения кода.

### F6. Привязка Steam ID и персональная статистика

- Ввод Steam ID / ссылки на профиль вручную ИЛИ автоопределение из GSI (`player.steamid`) при первом запуске игры.
- Загрузка из STRATZ: пул героев, винрейты по героям и матчапам, последние матчи.
- Без регистрации и пароля. Данные хранятся локально.

**Acceptance criteria:**
- [ ] Автоопределение Steam ID из первого GSI-пакета с подтверждением пользователя.
- [ ] Профиль обновляется после каждого завершённого матча.
- [ ] Приложение полностью функционально без привязки (кроме персональных фич — они помечаются как «привяжи Steam ID»).

## 4. Рекомендации по техническому стеку

| Слой | Технология | Обоснование |
|---|---|---|
| Платформа | **Electron** | Оверлей = прозрачное always-on-top окно (`transparent: true`, `frame: false`, `setAlwaysOnTop('screen-saver')`, `setIgnoreMouseEvents` для click-through) поверх Dota 2 в borderless |
| Язык | **TypeScript** | Строгая типизация GSI-пакетов и GraphQL-схем; идеален для AI-assisted разработки |
| UI | **React + Tailwind CSS** | Компонентная модель под 3 режима оверлея; быстрые итерации через Claude |
| GSI-приёмник | Встроенный **Node.js HTTP-сервер** (локальный, порт 3000+) | GSI шлёт POST JSON на localhost; конфиг-файл кладётся в `dota 2 beta/game/dota/cfg/gamestate_integration/` |
| Данные статистики | **STRATZ GraphQL API** | Бесплатный, богатые данные по матчапам и синергиям, фильтры по патчу и рангу; требуется API-токен |
| Резервный источник | OpenDota API | Fallback при недоступности STRATZ |
| Локальное хранилище | **SQLite** (better-sqlite3) + JSON-конфиги | Кэш статистики, профиль, настройки |
| Состояние приложения | Zustand или Redux Toolkit | Единый store: gameState + advice + settings |
| Обновления контента | Локальные JSON-конфиги | timings.json, rules.json, matchup-knowledge.json и др. редактируются напрямую, горячая перезагрузка без пересборки приложения |

**Ключевые ссылки для разработчика:**
- Electron BrowserWindow (transparent, always-on-top, click-through): https://www.electronjs.org/docs/latest/api/browser-window

## 5. Концептуальная модель данных

### 5.1 Локальные сущности (SQLite)

**UserProfile**
- `steam_id`: string (unique) — 64-bit Steam ID
- `verbosity`: enum('beginner','experienced') — default 'experienced'
- `hotkey_expanded_panel`: string — default 'F9'
- `overlay_positions`: JSON — позиции панелей
- `notifications_config`: JSON — вкл/выкл по типам событий
- `widgets_config`: JSON — набор, порядок и видимость статичных виджетов компактной панели
- `draft_ranking_mode`: enum('meta','personal') — default 'personal' при привязанном Steam ID, иначе 'meta'
- `created_at`, `updated_at`: timestamp

**HeroPoolStats** (кэш из STRATZ)
- `steam_id`: string (FK)
- `hero_id`: int
- `matches_count`: int, `winrate`: float
- `last_synced`: timestamp

**MatchupCache**
- `hero_id`: int, `other_hero_id`: int
- `relation`: enum('vs','with') — контрпик (против) или синергия (вместе)
- `winrate`: float, `sample_size`: int
- `patch`: string, `rank_bracket`: string
- `builds`: JSON (скиллбилд + предметы)
- `fetched_at`: timestamp (TTL 24 ч)

**MatchHistory** (для аналитики «до/после»)
- `match_id`: bigint, `hero_id`: int, `enemy_mid_hero_id`: int (nullable)
- `result`: enum('win','loss'), `kda`: JSON, `played_at`: timestamp

### 5.2 Контентные конфиги (локальные JSON, горячая перезагрузка)
- **timings.json:** `{ event_type, interval_sec | fixed_times[], warn_before_sec, patch }`
- **rules.json:** `{ rule_id, condition (DSL/JSON-logic), message_ru, priority (1–5), cooldown_sec, min_verbosity }`
- **matchup-knowledge.json:** `{ hero_id, vs_hero_id, do_tips[], avoid_tips[], power_spikes[], kill_windows[] }` — ключ всегда ПАРА героев; все тексты пишутся с позиции hero_id
- **hero-profiles.json:** `{ hero_id, ult_is_kill_window (bool), power_spike_levels (int[]), aggression_pattern (enum: all_in | trade | passive_farm), typical_level6_time_sec (int), notes }` — используется правилами F4 и статистическим fallback F2
- **gsi-field-catalog.json:** `{ field_path (напр. 'hero.health_percent'), label_ru, category (enum: hero | player | match | abilities | items), format (enum: int | percent | time | gold | bool | text), preset (bool — составной виджет вместо сырого поля) }` — каталог для конструктора виджетов F5; наполняется по ревизии GSI в M0
- **benchmarks.json:** `{ hero_id, minute, lh_p50, lh_p75, networth_p50, networth_p75, xp_p50, xp_p75, rank_bracket, patch }` — эталонные кривые для бенчмарк-виджетов; предрассчитываются из STRATZ/OpenDota, обновляются с патчами

### 5.3 Runtime-состояние (in-memory)
**GameState** — типизированное зеркало GSI-пакета: `map` (game_state, clock_time, daytime), `player` (steamid, gold, kills, deaths), `hero` (id, level, health_percent, mana_percent, buyback_cooldown, ult_status), `abilities`, `items`.

## 6. Принципы UI-дизайна

Наброски будут созданы через **Claude Design** на основе следующих принципов:

1. **Тёмная полупрозрачная тема** — оверлей не должен «светить» поверх игры; фон rgba(10,12,16,0.85), скругления, тонкие бордеры.
2. **Зоны, свободные от HUD Dota:** нельзя перекрывать миникарту (левый/правый нижний угол по настройке игрока), панель героя (низ по центру), топ-бар счёта. Дефолт компактной панели — верхний левый угол ниже топ-бара; уведомления — над панелью героя со смещением вверх.
3. **Иерархия текста:** одно уведомление = одна мысль ≤ 60 символов + опциональная строка деталей. Крупный тайминг, мелкий контекст.
4. **Цветовое кодирование:** зелёный — окно агрессии/возможность; жёлтый — тайминг/напоминание; красный — опасность/предупреждение.
5. **Ноль анимаций дольше 200 мс** — ничего не должно отвлекать в файте.
6. **Тон текста (уровень Archon–Ancient):** конкретика без банальностей. Не «будь осторожен», а «SF собрал бутылку — не разменивайся без своей».

## 7. Соображения по безопасности и легальности

1. **Только официальные каналы данных:** GSI (пассивное получение JSON от самой игры) + публичные API. Никакого чтения памяти процесса, никаких инъекций (оверлей — обычное прозрачное окно уровня ОС, процесс игры не затрагивается вообще), никакой автоматизации ввода и снятия тумана войны. Приложение только информирует — все действия совершает игрок.
2. **GSI-конфиг:** установка файла в папку Dota — штатный механизм Valve; выполняется приложением при первом запуске с подтверждением, с возможностью отката.
3. **Приватность:** Steam ID и статистика хранятся только локально; телеметрии нет.
4. **API-токен STRATZ:** персональный, хранится в env-переменной, не коммитится в репозиторий.
5. **Соответствие правилам STRATZ:** атрибуция («Powered by STRATZ») в UI, соблюдение rate limits, кэширование для минимизации запросов.

## 8. Этапы разработки / Milestones

### M0 — Стабилизация borderless, каркас и GSI (1–2 недели AI-assisted)
- **Шаг 0 (go/no-go, до кода):** стабилизация Dota 2 в borderless на целевой машине — Fullscreen Optimizations вкл/выкл, Vulkan vs DirectX 11, G-Sync/V-Sync, лимит FPS. Критерий: без фризов 2–3 матча подряд.
- Проект на Electron, TypeScript, React.
- Локальный HTTP-сервер GSI, установка gamestate_integration-конфига.
- Типизация GSI-пакетов, in-memory GameState store.
- Прозрачное click-through окно оверлея поверх Dota 2 в borderless, хоткей переключения интерактивности.
- Ревизия реального состава GSI-пакетов: данные драфта (видны ли пики обеих команд), полный список полей героя/предметов — фиксируется в `docs/gsi-fields.md`. От результата зависит объём ручного ввода в драфте.
- **Definition of Done:** borderless стабилен (без фризов в 2–3 матчах); панель с игровым временем из GSI видна поверх игры; составлен документ реально доступных GSI-полей.

### M1 — Тайминговые напоминалки и конструктор виджетов (F3, F5)
- timings.json, движок событий, всплывающие уведомления, компактная панель.
- Конструктор виджетов: gsi-field-catalog.json (по результатам ревизии GSI из M0), меню выбора полей с категориями и drag-and-drop порядком.
- Настройки вкл/выкл, позиционирование, тихий режим.
- **DoD:** полный матч с корректными напоминаниями о рунах/стаках, FPS-просадка ≤ 5%; любое поле из каталога добавляется на панель через меню и корректно обновляется в игре.

### M2 — STRATZ и помощник по пикам (F1)
- GraphQL-клиент, кэш SQLite, определение драфта через GSI.
- Панель контрпиков с винрейтами.
- **DoD:** в реальном драфте панель показывает актуальные контрпики ≤ 2 сек.

### M3 — Steam ID и персонализация (F6)
- Автоопределение Steam ID, синк пула героев, персональное ранжирование пиков.
- **DoD:** рекомендации учитывают личный пул и винрейты.

### M4 — План на лайн и бенчмарки (F2, F5 частично)
- Расширенная панель по хоткею, карточки матчапов (первые 30 героев), билды из STRATZ.
- Бенчмарк-виджеты: пайплайн генерации benchmarks.json из STRATZ/OpenDota, live-сравнение LH/networth/XP с нормой, цветовая индикация.
- Пресет позиции «Вместо стандартной панели» с калибровкой под разрешения.
- **DoD:** для 30 топ-мид-героев доступен полный план на лайн; бенчмарк-виджет корректно сравнивает показатели в реальном матче.

### M5 — Ситуативные подсказки (F4)
- advice-engine, rules.json, первые 10–15 правил, антиспам-логика.
- **DoD:** правила срабатывают корректно в 3 тестовых матчах без ложных спамов.

### M6 — Полировка
- Автозапуск приложения вместе с системой/игрой, обработка недоступности STRATZ (офлайн-режим на кэше), автообновление статистики после патчей.
- Удобное редактирование конфигов (timings, rules, hero-profiles, matchup-knowledge): валидация JSON при загрузке, горячая перезагрузка без перезапуска приложения.
- Итоговая настройка панелей и правил по опыту реальных матчей.

## 9. Потенциальные проблемы и способы их решения

| Проблема | Риск | Решение |
|---|---|---|
| Фризы Dota 2 в borderless на целевой машине | Высокий | Стабилизация в M0 до написания кода (Fullscreen Optimizations, Vulkan, G-Sync/V-Sync, лимит FPS); fallback — режим второго монитора |
| GSI не отдаёт данные о врагах (позиции, предметы) | Средний | Дизайн подсказок изначально строится на своих данных + таймингах; враждебный контекст берётся из статистики матчапа, а не из live-данных |
| Оценка уровня/ульты врага по таймингам неточна | Средний | Консервативная калибровка, явная маркировка «вероятно», возможность отключить оценочные подсказки в настройках |
| Патчи Dota ломают тайминги и мету | Высокий (регулярный) | Все тайминги/правила/знания — в редактируемых JSON-конфигах; проверка версии патча через STRATZ при старте; баннер «данные обновляются» после мажорного патча |
| Лимиты/недоступность STRATZ API | Средний | Кэш с TTL + прогрев топ-героев; graceful degradation до OpenDota; офлайн-режим на кэше с пометкой давности |
| Поминутные бенчмарки могут быть недоступны в API в нужной детализации | Средний | Проверка источников в M2; fallback — интерполяция из агрегатов (перцентили GPM/XPM за матч → расчётная кривая); бенчмарки честно помечаются как приблизительные |
| Перегруз оверлея → раздражение | Средний | Жёсткие лимиты уведомлений (≤1/30 сек ситуативных, ≤2 на экране), тихий режим, гранулярные настройки |
| Конфликт с другими оверлеями (Steam, Discord) | Низкий | Тест совместимости в M1; заметка в troubleshooting-гайд |
