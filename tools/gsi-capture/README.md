# GSI capture — сбор реальных GSI-пакетов для TASK-009

Набор для захвата сырых Game State Integration-пакетов на **Windows-машине с
Dota 2**. Репозиторий на той машине не нужен — достаточно скопировать эту папку
(`tools/gsi-capture/`) целиком, например на флешку или через облако.

Всё легально (INV3): Dota сама шлёт JSON POST'ом на localhost по штатному
механизму Valve; никакого чтения памяти или инъекций.

## Шаги на Windows-машине

1. **Скопировать cfg в Dota.** Файл `gamestate_integration_midmind_capture.cfg`
   положить в папку (создать `gamestate_integration`, если её нет):

   ```
   C:\Program Files (x86)\Steam\steamapps\common\dota 2 beta\game\dota\cfg\gamestate_integration\
   ```

2. **Включить GSI в лаунч-опциях.** Steam → ПКМ по Dota 2 → «Свойства…» →
   «Параметры запуска» → добавить:

   ```
   -gamestateintegration
   ```

   Без этого флага Dota пакеты не шлёт вообще.

3. **Запустить слушатель.** Двойной клик по `capture.cmd` (обёртка запускает
   `capture.ps1` со встроенным PowerShell — ставить ничего не нужно).
   Альтернатива при наличии Node ≥ 18: `node capture.mjs`.

   Проверка: открыть <http://localhost:3399/> в браузере — должно ответить `OK`.

4. **Поиграть с запущенным слушателем.** Нужно 2–3 сессии разных типов —
   каждая закрывает свой кусок ревизии TASK-009:

   - **обычный матч от начала** (важно застать стадию выбора героев —
     открытый вопрос: видны ли пики обеих команд на `HERO_SELECTION`);
   - **просмотр демо или спектейт** (секции `draft`/`buildings`/`couriers`/
     `roshan`/`minimap` могут приходить только в режиме наблюдателя — это
     тоже ответ, который нужен ревизии);
   - лобби с ботами тоже годится для быстрой проверки механики.

   В консоли на каждый пакет печатается строка с `game_state` и `clock_time` —
   так видно, что захват работает и какая фаза матча идёт.

5. **Остановить и забрать данные.** Ctrl+C (или закрыть окно) — пакеты пишутся
   на диск сразу, ничего не теряется. Папку `captured/` переименовать по типу
   сессии и перенести в репозиторий на dev-Mac:

   ```
   test/fixtures/gsi/raw/<имя-сессии>/     напр. raw/match-ranked-1/, raw/demo-spectate/
   ```

   Если пакетов много, папку можно заархивировать для переноса — главное
   сохранить исходные имена файлов (в них порядок и время прихода).

## Что дальше (на dev-Mac)

Как только дампы лежат в `test/fixtures/gsi/raw/`, TASK-009 полностью
выполнима без игры: анализ полей → `docs/gsi-fields.md` →
`content/gsi-field-catalog.json`, и вся цепочка зависимых задач
разблокируется.

## Примечания

- Порт слушателя — `3399` (не пересекается с GSI-сервером приложения, 3000+).
  Поменять: `capture.ps1 -Port NNNN` / `PORT=NNNN node capture.mjs` — тогда
  поправить порт и в cfg.
- В cfg запрошен **расширенный** набор секций (`draft`, `buildings`, `events`,
  `minimap`, …) — шире, чем боевой конфиг приложения (`buildGsiConfigContent.ts`).
  Это намеренно: цель ревизии — узнать, что реально приходит. Неизвестные/пустые
  секции Dota просто игнорирует.
- Приватность: пакеты содержат `player.steamid` и имя аккаунта. Для локальных
  фикстур это нормально, но помни об этом, если репозиторий станет публичным.
- `auth.token` в cfg — `midmind-capture`; слушатель токен не проверяет (это
  временный локальный инструмент), но токен останется в сохранённых пакетах —
  боевого секрета в нём нет.
